# Challenge 09 - Yay or Nay

The purpose of this challenge is to practice lists at an introductory level, adding gesture controlled UI, and practicing with timings.

## Mild 🌶️

Create a program that meets the following criteria:

* A live video feed is captured throug a webcam.
* Hands are detected.
* Hands are drawn.
* There are a list of topics/words that a person could express a "like" or "dislike" opinion on.
* The program displays a word from the list.
* The human can either show a thumbs up or a thumbs down.
* The program waits to determine that a consistant answer has been given for at least one second.
* Once the program has determined a thumbs up or thumbs down for a full second, it displays a message in text about if the human has voted "yay" or "nay" on the topic.
* The program moves onto the next word in the list and continues to repeat the process until the list is complete.

## Spicy 🌶️🌶️🌶️

Create a program that meets the following criteria:

* A live video feed is captured throug a webcam.
* Hands are detected.
* Hands are drawn.
* The left side of the screen has a rectangle labeled "Yay"
* The right side of the screen has a rectangle labeled "Nay"
* There are a list of topics/words that a person could express a "like" or "dislike" opinion on.
* The program displays a word from the list.
* The human can either show a thumbs up or a thumbs down.
* The program waits to determine that a consistant answer has been given for at least one second.
* Add a UI element such as an arc that shrinks or a progress bar with a rectangle to show how long the program has left until it has confirmed the answer.
* Once the program has determined a thumbs up or thumbs down for a full second, it displays a message in text about if the human has voted "yay" or "nay" on the topic.
* The program adds the word to the appropriate rectangle to maintain a list of how the human player has voted.
* The program moves onto the next word in the list and continues to repeat the process until the list is complete.
